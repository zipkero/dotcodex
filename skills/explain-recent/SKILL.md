# Explain Recent

## 목적

직전 작업에서 발생한 수정, 삭제, 생성의 이유와 역할, 전체 흐름을 쉽고 상세하게 설명한다. 설명은 실제 근거에 기반하고, 확인되지 않은 의도는 추정으로 분리한다.

## 확인 순서

1. 사용자가 설명 범위를 지정했으면 그 범위를 우선한다. 지정하지 않았으면 최근 대화, 실행한 명령, 편집한 파일, `git diff` 또는 관련 산출물을 확인해 직전 작업 범위를 파악한다.
2. 변경을 `수정`, `삭제`, `생성`, `검증`, `보류/미처리`로 분류한다.
3. 각 변경의 역할을 사용자 목표와 연결해 설명한다.
4. 작업 흐름을 시간순 또는 의존성 순서로 정리한다.
5. 근거가 부족한 부분은 단정하지 말고 “확인 가능한 범위에서는”, “추정하면”처럼 표시한다.

## 설명 방식

- 먼저 한 문단으로 결론을 말하되, 결론만으로 끝난 것처럼 보이지 않게 이후 섹션에서 근거와 흐름을 확장한다.
- 이어서 사용자가 기억해야 할 핵심을 `변경 전에는 무엇이 문제였고`, `이번 변경으로 무엇이 가능해졌는지`의 대비로 보여준다.
- 변경된 파일과 변경 유형을 정리하되, 한 줄 요약으로 끝내지 않는다.
- 각 변경마다 다음 세 가지를 포함한다.
  - `이유`: 왜 필요했는지
  - `역할`: 전체 작업에서 어떤 책임을 맡는지
  - `흐름`: 앞뒤 작업과 어떻게 이어지는지
- 쉽고 상세한 설명을 기본값으로 삼는다. 전문 용어는 필요한 만큼 풀어 설명하고, 사용자가 판단할 수 있는 맥락을 생략하지 않는다.
- 상세함은 단순한 분량이 아니라 판단 근거의 충분함이다. 요구사항 배경, 변경 전 문제, 변경 후 보장되는 동작, 검증이 의미하는 바를 각각 설명한다.
- 설명은 사용자가 머릿속으로 실행 흐름을 그릴 수 있게 쓴다. 코드 구조 나열보다 “입력/상황이 들어오면 어떤 코드가 어떤 순서로 반응하고, 결과가 어떻게 달라지는지”를 우선한다.
- 추상적인 표현만 반복하지 말고, 실제 값, 상태, 함수명, sequence, 요청/응답 예시처럼 관찰 가능한 단서를 사용한다.
- 삭제와 생성도 단순히 “삭제/생성했다”로 끝내지 말고, 삭제된 책임이나 새로 생긴 책임이 전체 흐름에서 어떤 의미인지 설명한다.
- 코드 구현 세부사항보다 사용자가 판단해야 할 요구사항, 영향, trade-off를 먼저 설명한다.
- 검증을 했다면 어떤 명령이나 확인으로 성공을 판단했는지 함께 말한다.
- 검증하지 못했거나 아직 남은 위험이 있으면 마지막에 별도로 밝힌다.

## 출력 형태

항상 다음 구조를 사용한다. 해당 항목이 없으면 `없음` 또는 `확인하지 않음`으로 표시한다.

~~~markdown
한눈 결론:
- 이번 작업의 핵심:
- 머리에 남길 변화:

배경:
- 이 작업이 필요했던 사용자 목표나 요구사항:
- 변경 전 문제가 되거나 비어 있던 지점:
- 이번 작업으로 보장하려는 동작:

변경 전/후:
| 구분 | 변경 전 | 변경 후 | 왜 중요한가 |
| --- | --- | --- | --- |
| ... | ... | ... | ... |

변경 내역:
### `path`
- 변경 유형: 수정/삭제/생성
- 이유:
- 역할:
- 흐름:
- 변경 후 의미:

변경 근거(diff):
```diff
...
```
위 diff는 변경 이유, 역할, 흐름을 판단하는 데 필요한 hunk만 포함한다. 생략한 diff가 있으면 생략 범위와 이유를 밝힌다.

diff 해설:
- 이 diff에서 중요한 줄:
- 그 줄이 동작이나 책임을 어떻게 바꾸는지:
- 놓치기 쉬운 영향:

실제 흐름:

3. ...
위 흐름은 사용자가 실제로 겪는 상황이나 테스트가 재현한 상황을 기준으로 쓴다.

검증:
- 실행한 검증:
- 검증이 확인한 것:
~~~

## 주의사항

- 직전 작업 밖의 변경이나 사용자가 만든 변경을 내 작업처럼 설명하지 않는다.
- 관련 없는 리팩터링, 취향, 일반론을 설명에 섞지 않는다.
- 파일을 새로 수정하지 말고, 설명 요청으로 처리한다.
- diff는 파일 링크보다 우선 사용하되, 전체 파일을 길게 붙이지 말고 설명에 필요한 변경 부분만 사용한다.
- 사용자가 원인을 묻더라도 실제 diff나 로그로 확인되지 않으면 추정이라고 표시한다.
