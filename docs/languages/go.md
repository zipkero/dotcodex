# Go 작업 기준

## 적용 범위
- 이 문서는 전역 기본값이다. 프로젝트의 `AGENTS.md`, `go.mod`, formatter, linter, test 설정이 있으면 그 기준을 우선한다.
- Go 파일을 수정하거나 검토할 때 적용한다.

## 코드 작성
- `gofmt` 또는 `go fmt` 결과를 기준으로 삼는다.
- 기존 프로젝트가 달리 정하지 않으면 표준 라이브러리와 작은 명시적 구현을 우선한다.
- 공개 API 변경은 호출부, 테스트, 문서 영향을 함께 확인한다.
- 패키지 경계는 역할 중심으로 유지하고, 단순한 변경을 위해 새 추상화를 만들지 않는다.

## 오류와 동시성
- 오류는 숨기지 말고 호출자가 판단할 수 있는 context와 함께 반환한다.
- `panic`은 초기화 실패나 복구 불가능한 프로그래밍 오류에만 제한한다.
- goroutine을 추가하거나 수정할 때는 종료 조건, cancellation, channel close 책임을 확인한다.
- `context.Context`가 이미 흐르는 경로에서는 cancellation과 timeout 전달을 끊지 않는다.

## 테스트
- 기존 테스트 관례를 따른다.
- 순수 로직은 table-driven test를 우선 고려한다.
- 프로젝트 관례가 없고 Go 코드 동작을 바꿨다면 `go test ./...`를 우선 검증 명령으로 고려한다.

## 주석
- 새 패키지에는 패키지의 역할과 사용 범위를 설명하는 패키지 주석을 작성한다.
  기존 패키지의 역할이 바뀌면 패키지 주석도 함께 갱신한다.
- 패키지 주석은 패키지당 한 파일에서 `package` 선언 바로 앞에 배치한다.
- 새로 추가하는 top-level exported identifier와 메서드에는 Go doc 주석을 작성한다.
  기존 exported identifier를 수정할 때 문서 주석이 없거나 호출자 계약이 달라지면 같은 변경에서 보완한다.
- 패키지와 선언의 Go doc 주석은 선언의 행 끝에 붙이지 않고 빈 줄 없이 선언 바로 앞에 배치한다.
- 개별 exported identifier의 문서 주석은 식별자 이름으로 시작하고, 대상이 나타내거나 제공하는 내용을 완결된 문장으로 설명한다.
- `const`와 `var` 선언 그룹의 주석은 그룹 전체의 관계와 용도를 설명할 수 있으며 개별 식별자 이름으로 시작하지 않아도 된다.
- 타입 주석에는 인스턴스가 나타내는 대상과 책임을 설명한다.
  호출자가 알아야 하는 경우 `zero value`의 의미, 초기화 조건과 동시성 안전성도 함께 설명한다.
- 함수와 메서드 주석에는 호출자가 알아야 하는 동작, 반환값, 부수 효과, 오류 조건과 사용 조건을 설명한다.
- exported struct field의 의미가 타입 주석에서 충분히 설명되지 않으면 필드별 주석을 작성한다.
- struct field와 `const` 또는 `var` 그룹 원소의 짧은 설명은 프로젝트 관례에 따라 행 끝 주석으로 작성할 수 있다.
- `오류와 동시성` 절에서 확인한 조건 중 호출자가 알아야 하는 내용은 관련 선언의 문서 주석에 반영한다.
